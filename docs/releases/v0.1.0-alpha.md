# v0.1.0-alpha

发布日期：待定

## Highlights

- 开源主链路已经收敛为 `frontend/ + we-mp-rss/`
- 官网来源聚合可以独立运行，不再默认依赖公众号链路
- 前端生产环境默认关闭演示数据兜底，后端故障会展示真实状态
- 新增来源状态、来源统计、队列状态、配置摘要和健康检查接口
- Docker 运行方式已经统一到 `START_MODE=api|worker|auth|all`
- 启动时支持自动建表和管理员 bootstrap

## Breaking / Config Changes

- 旧的 `-job/-init` 启动方式不再是推荐入口，统一改用：
  - `python3 main.py -config config.yaml --mode api|worker|auth|all`
- Docker 入口同样使用 `START_MODE=api|worker|auth|all`
- 管理员初始化不再使用旧的 `USERNAME/PASSWORD` 假参数，当前正式入口只有：
  - `ADMIN_USERNAME`
  - `ADMIN_PASSWORD`
  - `ADMIN_PASSWORD_HASH`
  - `ADMIN_FORCE_UPDATE_PASSWORD`
- 前端构建产物必须先生成到 `we-mp-rss/static/calendar/`，否则根路径会明确返回 `503 frontend_not_built`

## Validation

本版本发布前已运行：

- `python3 -m unittest discover -s we-mp-rss/tests`
- `cd frontend && npm run test:smoke`
- `cd frontend && npm run lint`
- `cd frontend && npm run build`

## Deployment Notes

- 最小部署：`--mode api`
- 本机接通公众号维护链路：`--mode all`
- API + worker 分离：需要共享 Redis
- Docker 已提供：
  - 单容器模式
  - SQLite 最小示例
  - API + worker 分离模式

## Known Issues

- 公众号链路仍然是可选增强能力，不保证长期稳定
- PostgreSQL 路径已经补文档，但当前仍缺正式迁移体系
- 管理员维护面板尚未完成
- 来源 fixture 回放测试、调试快照导出、活动屏蔽机制尚未完成

## Scope

本次开源 alpha 只覆盖主产品仓库内容：

- `frontend/`
- `we-mp-rss/`

不包含本地 sidecar 工具目录 `wexin-read-mcp/`
